## Музыкальный сервис персональных песен (ProSound) 1.0

---

## 1. ОБЩЕЕ ОПИСАНИЕ ПРОЕКТА

Проект — это **современный музыкальный сервис**, основной продукт которого — **персональные песни на заказ**.

Сервис сочетает:

- каталог реальных песен,
- истории клиентов,
- полноценный аудиоплеер,
- подарочную механику,
- заказ персональной песни как логичное продолжение эмоции.

Проект НЕ является:

- обычным лендингом,
- каталогом аудиофайлов,
- маркетинговой страницей.

Это **музыкальное приложение**, ориентированное на эмоцию, доверие и пользовательский опыт.

---

## 2. АРХИТЕКТУРА ПРИЛОЖЕНИЯ

### 2.1 Общая модель

- One App Application (SPA / App Shell)
- Client-side routing
- Глобальное состояние (плеер, тема, пользователь)
- Аудиоплеер НЕ зависит от роутов
- Быстрая навигация без перезагрузок

### 2.2 App Shell

- Меню
- Глобальный плеер

- Основной контент

App Shell монтируется один раз и не размонтируется.

## 3. ГЛОБАЛЬНЫЙ АУДИО-ПЛЕЕР

### 3.1 Функциональность

- Play / Pause
- Seek
- Очередь
- Плейлисты
- Shuffle / Repeat
- Работа в фоне при навигации

### 3.2 UX и визуал

### 3.3 Обязательные элементы плеера

### 3.4 Запреты

- Кнопка «Enable playback» УБРАНА
- Никаких лишних действий для запуска

## 4. ДИНАМИЧЕСКИЙ ГЛОБАЛЬНЫЙ ФОН

### 4.1 Поведение

- Фон берётся из обложки текущего трека (currentTrack.coverUrl)
- Обложка:
  - размывается (blur)
  - слегка затемняется
- При смене трека фон **плавно crossfade-ится**

### 4.2 Архитектура

- Отдельный компонент <GlobalBackdrop />
- Живёт в App Shell
- Подписан ТОЛЬКО на:
  - currentTrack
  - theme
- Не зависит от роутов

### 4.3 Технические требования

- Использовать 2 слоя фона (A / B)
- Анимируется ТОЛЬКО opacity
- Предзагрузка следующей обложки
- Учитывать prefers-reduced-motion

## 18. ПОВЕДЕНИЕ ПРИ ПЕРВОМ ЗАХОДЕ (INITIAL EXPERIENCE)

### 18.1 Автозагрузка последнего трека (ОБЯЗАТЕЛЬНО)

При первом заходе пользователя на сайт:

- Система автоматически:
  - запрашивает **последний опубликованный трек**
  - устанавливает его как currentTrack в Player Store
- Плеер:
  - **уже содержит трек**
  - готов к воспроизведению сразу
- Пользователь может нажать **Play без дополнительных действий**

❌ Запрещено:

- Tap to enable audio
- Enable playback
- любые промежуточные кнопки

⚠️ Допустимо:

- воспроизведение начинается **по первому пользовательскому действию** (клик Play),
   но **без дополнительных UX-барьеров**

---

### 18.2 Инициализация динамического фона

- Сразу после загрузки currentTrack:
  - <GlobalBackdrop /> получает coverUrl
  - фон:
    - размывается
    - затемняется
    - отображается **даже до нажатия Play**

📌 Результат:
 Сайт **с первого экрана выглядит живым и музыкальным**, а не пустым.
